# Definition Of Done

## Local Non-Server Initialization

Done when:

- project has README, AGENTS, CLAUDE, active TODO, milestones, research plan,
  paper claim status, experiment protocol, baseline contract, data/eval plan,
  motivation/ablation/hparam plan, risks/blockers, server runbook, configs,
  validator, tests, git commit, and GitHub remote;
- local validator passes;
- local tests pass;
- git status is clean after commit/push;
- server tasks are blocked by explicit runbook gates, not by missing planning.

## Server Phase

Done only when:

- ARIS experiment-plan review passes;
- exact command is approved;
- lightweight artifacts are synced back;
- experiment-audit labels evidence tiers;
- paper claims are updated to match evidence.

## Paper Phase

Done only when:

- citation audit has no must-add gaps;
- paper claim audit is READY or READY WITH EDITS;
- no unsupported central claim remains.
